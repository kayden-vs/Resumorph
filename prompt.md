# Coding Agent Prompt: Resume Tailor (Go)

## Project Summary

Build a CLI tool in Go called `resume-tailor` that tailors a LaTeX resume to a job description.
It works by scanning a pre-annotated `.tex` template for `%%KEY%%` placeholders, calling the
Gemini Flash LLM API to rewrite the content values for a given job description, injecting the
new values back into the template, and compiling the result to PDF via `pdflatex`.

## Conventions That Apply to Every File

- All errors must be wrapped with context using `fmt.Errorf("context: %w", err)` before
  returning or printing.
- Fatal errors: print `Error: <message>` to `os.Stderr` then call `os.Exit(1)`.
- Success messages: print to `os.Stdout`.
- Every exported function in `cmd/` has this signature:
  `func CommandName(args []string)` where `args` is `os.Args[2:]`.
- Each command uses `flag.NewFlagSet(name, flag.ExitOnError)` — never the global `flag`
  package — so flags are scoped per subcommand.
- Marker format: `%%KEY%%` where KEY is uppercase letters, digits, and underscores only.
  The regex to match them everywhere is: `%%([A-Z0-9_]+)%%`

## main.go

### Purpose
Entry point. Parses the subcommand from `os.Args[1]` and dispatches to the right function
in `cmd/`. Also handles `--help` and unknown commands.

### Behaviour

```
Usage: resume-tailor <command> [flags]

Commands:
  extract   Scan template.tex for %%KEY%% markers, write skeleton content.json
  tailor    Call Gemini API to rewrite content.json for a job description
  apply     Inject modified JSON values into template.tex, write output.tex
  compile   Run pdflatex on output.tex to produce output.pdf
  run       Run tailor → apply → compile in sequence (full pipeline)
```

### Implementation

```go
package main

import (
    "fmt"
    "os"
    "resume-tailor/cmd"
)

func main() {
    if len(os.Args) < 2 {
        printUsage()
        os.Exit(1)
    }

    switch os.Args[1] {
    case "extract":
        cmd.Extract(os.Args[2:])
    case "tailor":
        cmd.Tailor(os.Args[2:])
    case "apply":

## cmd/extract.go

### Purpose
Scans `template.tex` for every `%%KEY%%` placeholder. Writes `content.json` — a flat
JSON object where each key maps to an empty string `""`. The user fills in their actual
resume content values into this file once. After that, this command is not needed again.

### Flags
| Flag | Default | Description |
|------|---------|-------------|
| `--template` | `template.tex` | Path to the annotated LaTeX template |
| `--output` | `content.json` | Path to write the skeleton JSON |

### Detailed Logic

1. Parse flags from `args`.
2. Read the template file with `os.ReadFile`.
3. Compile the regex `%%([A-Z0-9_]+)%%`.
4. Find all matches using `re.FindAllStringSubmatch(content, -1)`.
5. Deduplicate keys by inserting into a `map[string]bool`.
6. Build a `map[string]string` with every unique key mapped to `""`.
7. If `content.json` already exists at the output path, print a warning:
   `Warning: output file already exists and will be overwritten. Press Enter to continue or Ctrl+C to cancel.`
   Then call `fmt.Scanln` to wait for confirmation before proceeding.
8. Marshal the map with `json.MarshalIndent(data, "", "  ")`.
9. Write to the output file.
10. Print summary:
    ```
    ✓ Found 23 unique markers in template.tex
    ✓ Written skeleton to content.json
    
    Next step: open content.json and fill in your current resume values for each key.
    Then run: resume-tailor tailor --jd <job_description.txt>
    ```

### Edge Cases
- If the template file does not exist: fatal error `template file not found: template.tex`.
- If zero markers are found: fatal error `no %%KEY%% markers found in template.tex —
  have you annotated your template?`
- Keys must be collected preserving no order guarantees. The JSON output key order does
  not matter.

## cmd/tailor.go

### Purpose
Reads `content.json` (your filled-in resume values), reads a job description text file,
calls the Gemini Flash API, gets back a modified JSON with the same keys but rewritten
values targeted at the job, and writes it to `modified.json`.

### Flags
| Flag | Default | Description |
|------|---------|-------------|
| `--content` | `content.json` | Path to filled content JSON |
| `--jd` | *(required)* | Path to plain-text job description file |
| `--output` | `modified.json` | Path to write the LLM-modified JSON |
| `--api-key` | `""` | Gemini API key. If empty, read from env var `GEMINI_API_KEY` |
| `--model` | `gemini-2.0-flash` | Gemini model name |

### Gemini API Details

**Endpoint:**
```
POST https://generativelanguage.googleapis.com/v1beta/models/{MODEL}:generateContent?key={API_KEY}
```

**Request body (Go struct to marshal):**
```go
type geminiRequest struct {
    Contents         []geminiContent  `json:"contents"`
    GenerationConfig generationConfig `json:"generationConfig"`
}

type geminiContent struct {
    Parts []geminiPart `json:"parts"`
}

type geminiPart struct {
    Text string `json:"text"`
}

type generationConfig struct {
    Temperature      float64 `json:"temperature"`
    ResponseMimeType string  `json:"responseMimeType"`
}
```

Set `temperature` to `0.3` and `responseMimeType` to `"application/json"`.

**Response body (Go struct to unmarshal):**
```go
type geminiResponse struct {
    Candidates []struct {
        Content struct {
            Parts []struct {
                Text string `json:"text"`
            } `json:"parts"`
        } `json:"content"`
    } `json:"candidates"`
    Error *struct {
        Message string `json:"message"`
        Code    int    `json:"code"`
    } `json:"error"`
}
```

Extract the response text from `response.Candidates[0].Content.Parts[0].Text`.

### The LLM Prompt

Build the prompt by string concatenation. The prompt must be exact:

```
You are an expert resume writer and career coach. Your task is to tailor a resume's content to match a specific job description, making it more likely to pass ATS screening and impress human reviewers.

You will receive:
1. A JSON object representing the current resume content, where each key is a section/field identifier and each value is the current text for that field.
2. A job description to tailor the resume toward.

Your job is to rewrite the VALUES in the JSON to better match the job description. Follow these rules strictly:

RULES:
- Return ONLY a valid JSON object. No explanation, no markdown, no code fences, no preamble.
- Keep EVERY key from the input JSON. Do not add new keys. Do not remove keys.
- Preserve all LaTeX formatting commands in the values (e.g., \textbf{}, \href{}, \textit{}, bullet content, etc.). Only change the human-readable text portions.
- Do not change values for fields that are factual and non-tailorable (e.g., your name, email, phone, address, LinkedIn URL, GitHub URL, GPA, degree name, university name, graduation date). You can identify these by their key names or because they contain contact/identity information.
- For bullet points describing work experience or projects, rewrite them to emphasize skills, tools, and outcomes that are most relevant to the job description. Use strong action verbs.
- For the skills section, reorder or slightly adjust emphasis to front-load skills mentioned in the job description. Do not fabricate skills that are not present.
- For the summary or objective (if present), rewrite it to directly speak to the role.
- Keep bullet point counts the same — do not add or remove bullets.
- Values must remain reasonable in length. Do not expand a short bullet into a paragraph.

CURRENT RESUME CONTENT JSON:
%s

JOB DESCRIPTION:
%s

Return the tailored JSON now:
```

Where `%s` placeholders are replaced with: (1) the pretty-printed content JSON string and
(2) the raw job description text.

### Detailed Logic

1. Parse flags.
2. Resolve API key: use `--api-key` flag first; if empty, read `os.Getenv("GEMINI_API_KEY")`;
   if still empty, fatal error:
   `Error: Gemini API key not provided. Use --api-key or set GEMINI_API_KEY env var.`
3. Read and unmarshal `content.json` into `map[string]string`. Fatal if file not found or
   invalid JSON.
4. Read job description file into a string. Fatal if file not found.
5. Re-marshal content JSON to indented string for prompt insertion.
6. Build the prompt string.
7. Build the `geminiRequest` struct.
8. Marshal to JSON bytes.
9. Create `http.NewRequest("POST", url, bytes.NewReader(body))`.
10. Set header `Content-Type: application/json`.
11. Execute with `http.DefaultClient.Do(req)`. Set a timeout: create a client with
    `Timeout: 60 * time.Second`.
12. Read response body with `io.ReadAll`.
13. If HTTP status is not 200, unmarshal the error field and fatal:
    `Error: Gemini API error (HTTP <status>): <error.message>`
14. Unmarshal into `geminiResponse`.
15. Extract text from `Candidates[0].Content.Parts[0].Text`. If candidates is empty,
    fatal: `Error: Gemini returned empty response. Try again.`
16. Strip any accidental markdown code fences from the text:
    - Trim leading/trailing whitespace.
    - If it starts with ` ```json ` or ` ``` `, strip the first line and the last line
      if it's ` ``` `.
17. Unmarshal the cleaned text into `map[string]string` as `modifiedContent`.
18. **Validation — this is critical:**
    - For every key in the original `content.json`, check it exists in `modifiedContent`.
      If any key is missing, fatal:
      `Error: LLM response is missing key %q — the model did not follow instructions.
       Try running tailor again.`
    - For every key in `modifiedContent`, check it exists in the original. If extra keys
      are present, print a warning `Warning: LLM added unexpected key %q — it will be
      ignored.` and delete it from `modifiedContent`.
19. Write `modifiedContent` as indented JSON to `--output` file.
20. Print:
    ```
    ✓ Gemini tailored your resume content successfully
    ✓ Written to modified.json
    
    Next step: resume-tailor apply
    ```

## cmd/apply.go

### Purpose
Takes `template.tex` (with `%%KEY%%` markers) and `modified.json`, replaces every marker
with its value, and writes `output.tex`. Also warns about any unresolved markers.

### Flags
| Flag | Default | Description |
|------|---------|-------------|
| `--template` | `template.tex` | Path to annotated template |
| `--content` | `modified.json` | Path to JSON with replacement values |
| `--output` | `output.tex` | Path to write the filled-in LaTeX file |

### Detailed Logic

1. Parse flags.
2. Read the template file.
3. Read and unmarshal the content JSON into `map[string]string`.
4. Start with `result := templateContent` as a string.
5. For each `key, value` in the map, do:
   `result = strings.ReplaceAll(result, "%%"+key+"%%", value)`
6. After all replacements, scan `result` for any remaining `%%([A-Z0-9_]+)%%` matches.
   For each found: print `Warning: marker %%<KEY>%% was not replaced — no value found in
   JSON.` (This catches keys in the template that the user forgot to add to content.json.)
7. Write `result` to the output file.
8. Print:
    ```
    ✓ Applied 23 replacements to template.tex
    ✓ Written to output.tex
    
    Next step: resume-tailor compile
    ```
   (Count is the number of keys in the map that were actually present in the template.)

### Edge Cases
- If a value in the JSON contains a backslash followed by n (`\n`), do not convert to
  newline. LaTeX handles line breaks — treat all values as raw strings. Use `strings.ReplaceAll`
  directly, not any format-parsing function.
- Do not trim whitespace from values. Preserve exactly what the LLM returned.

## cmd/compile.go

### Purpose
Runs `pdflatex` on `output.tex` to produce `output.pdf`. Runs pdflatex **twice** (required
for correct rendering of certain LaTeX features like section numbering and references).
Streams output live.

### Flags
| Flag | Default | Description |
|------|---------|-------------|
| `--input` | `output.tex` | Path to the .tex file to compile |
| `--output-dir` | `.` | Directory where PDF and aux files will be written |
| `--runs` | `2` | Number of pdflatex passes (default 2) |
| `--quiet` | `false` | Suppress pdflatex stdout (errors still shown) |

### Detailed Logic

1. Parse flags.
2. Check that `pdflatex` is available by running `exec.LookPath("pdflatex")`. If not found,
   fatal with:
   ```
   Error: pdflatex not found in PATH.
   Install a LaTeX distribution:
     macOS:   brew install --cask mactex  (or basictex)
     Ubuntu:  sudo apt-get install texlive-latex-base
     Windows: https://miktex.org/download
   ```
3. Check that the input file exists with `os.Stat`. Fatal if not found.
4. Run pdflatex in a loop `--runs` times (default 2). For each run:
   a. Build args: `[]string{"-interaction=nonstopmode", "-output-directory", outputDir, inputFile}`
   b. Create `cmd := exec.Command("pdflatex", args...)`
   c. If `--quiet` is false, pipe stdout to `os.Stdout` by setting `cmd.Stdout = os.Stdout`.
      Always set `cmd.Stderr = os.Stderr`.
   d. Print `Running pdflatex (pass <N>/<total>)...`
   e. Run with `cmd.Run()`. 
   f. If the command fails (non-zero exit), print:
      ```
      Error: pdflatex failed on pass <N>. Check the output above for LaTeX errors.
      Tip: Look for lines starting with "!" in the pdflatex output.
      ```
      Then `os.Exit(1)`.
5. After all passes succeed, determine the output PDF path:
   - Take the base name of the input file (e.g., `output.tex` → `output`).
   - Combine with `--output-dir`: `filepath.Join(outputDir, baseName+".pdf")`.
   - Verify the PDF exists with `os.Stat`. If somehow missing, print a warning.
6. Print:
    ```
    ✓ Compilation successful (2 passes)
    ✓ PDF written to ./output.pdf
    ```

## cmd/run.go  ← ADD THIS FILE (not in original structure, but needed for `run` command)

Wait — the original structure doesn't include `run.go`. Add it as `cmd/run.go`.

### Purpose
Chains `tailor → apply → compile` as a single command. Does not re-run `extract` (that's
a one-time setup step).

### Flags
Accepts a union of all flags from `tailor`, `apply`, and `compile`. Internally it calls
each function's logic by refactoring shared logic into unexported helper functions.

**Important implementation note:** Do NOT call `cmd.Tailor(args)`, `cmd.Apply(args)`,
`cmd.Compile(args)` directly from `Run` because they call `os.Exit` on failure. Instead,
extract the core logic of each command into unexported functions that return `error`, and
call those from both the public `CommandName(args []string)` entry points and from `Run`.

### Refactoring pattern for each cmd file:

```go
// Public entry point (called from main.go)
func Tailor(args []string) {
    if err := runTailor(args); err != nil {
        fmt.Fprintf(os.Stderr, "Error: %v\n", err)
        os.Exit(1)
    }
}

// Internal implementation that returns error (called from Run)
func runTailor(args []string) error {
    // all the logic here, return errors instead of os.Exit
}
```

Apply the same pattern to `Extract`, `Apply`, `Compile`.

### run.go Flags
| Flag | Default | Description |
|------|---------|-------------|
| `--jd` | *(required)* | Job description file path |
| `--api-key` | `""` | Gemini API key |
| `--model` | `gemini-2.0-flash` | Gemini model |
| `--content` | `content.json` | Input content JSON |
| `--template` | `template.tex` | LaTeX template |
| `--output-tex` | `output.tex` | Intermediate .tex output |
| `--modified` | `modified.json` | Intermediate modified JSON |
| `--output-dir` | `.` | PDF output directory |
| `--quiet` | `false` | Suppress pdflatex output |

### run.go Logic

1. Parse flags.
2. Print `[1/3] Tailoring content with Gemini...`
3. Call `runTailor` with constructed args. Fatal on error.
4. Print `[2/3] Applying content to template...`
5. Call `runApply` with constructed args. Fatal on error.
6. Print `[3/3] Compiling PDF...`
7. Call `runCompile` with constructed args. Fatal on error.
8. Print:
   ```
   ✓ Done! Your tailored resume is ready at ./output.pdf
   ```

## Important Implementation Notes

1. **JSON key order in output files:** Use `encoding/json` with `json.MarshalIndent`. Key
   order in maps is not guaranteed in Go — this is fine. The user reads the JSON by key
   name, not position.

2. **The marker regex must be compiled once and reused**, not recompiled on every call.
   Use `var markerRe = regexp.MustCompile(`%%([A-Z0-9_]+)%%`)` as a package-level var
   in each file that uses it.

3. **LaTeX special characters in values:** The LLM may occasionally return `&`, `%`, `_`,
   `#` etc. unescaped in contexts where LaTeX would choke. Do NOT auto-escape these —
   the user's original content JSON already has properly-escaped LaTeX, and the LLM is
   instructed to preserve LaTeX formatting. Escaping blindly would double-escape.

4. **HTTP client:** Do not use `http.Get` or `http.Post`. Always construct a proper
   `http.Client{Timeout: 60 * time.Second}` and use `client.Do(req)` so the timeout
   is respected.

5. **Reading the API response:** Always `defer resp.Body.Close()` immediately after
   checking the error from `client.Do`. Use `io.ReadAll(resp.Body)` to get the full
   response body, even on non-200 responses (needed for error message parsing).

6. **pdflatex output directory:** pdflatex places all output files (`.aux`, `.log`, `.pdf`)
   in `--output-directory`. If the directory doesn't exist, `pdflatex` will error. Add a
   check: if `outputDir != "."`, call `os.MkdirAll(outputDir, 0755)` before running.

7. **Counting replacements in apply:** Track which keys from the JSON were actually found
   and replaced in the template. A key present in JSON but with no matching marker in the
   template is just unused — log it at debug level but don't error.
